# BabyTracker

微信宝宝喂养记录小程序，当前已支持：

- 首页快捷记录、时间轴、记录编辑
- 统计页、里程碑、提醒、家庭信息
- 本地存储
- 微信云开发同步骨架和可部署云函数

## 云开发接入步骤

1. 在微信云开发控制台创建环境。
2. 把 `utils/config.js` 中的 `CLOUD_ENV_ID` 改成你的环境 ID。
3. 在微信开发者工具中右键 `cloudfunctions/babyTracker`，执行“上传并部署：云端安装依赖”。
4. 在云开发数据库中创建集合 `bt_families` 和 `bt_members`。
5. 重新编译小程序。
6. 进入“我的”页，使用“初始化云端 / 上传本地 / 拉取云端”验证同步链路。
7. 使用“邀请口令”和“输入邀请码加入家庭”验证双人协作。

## 当前云同步策略

- 小程序页面仍以本地缓存为直接数据源，保证页面响应速度。
- 应用启动时会尝试从云端拉取并覆盖本地缓存。
- 本地写操作后会自动异步上传整份家庭数据到云端。
- 如果云端失败，会自动回退到本地模式，并在“我的”页展示错误信息。

## 当前限制

- 里程碑媒体会优先上传到云存储，但暂时还没有做媒体删除回收。
- 家庭协作已通过邀请码共享家庭数据，成员映射保存在 `bt_members` 集合中。
- 目前使用整份 store 同步，适合 MVP；后续可拆成更细粒度的记录、提醒、里程碑集合。
